# 项目计划书：基于IMU的人体活动识别

## 一、项目背景

本项目旨在利用可穿戴传感器（ESP32 + MPU6050）采集人体运动数据，通过模式识别方法实现活动识别，重点关注跌倒检测。项目强调“用数字说话”的验收文化和严格的数据治理（按被试划分、防止数据泄漏）。

## 二、项目目标

| 目标编号 | 具体内容 | 量化指标 |
|---------|---------|---------|
| G1 | 跌倒检测 | 召回率 ≥ 95%，精确率 ≥ 90% |
| G2 | 活动分类（走路/跑步/静坐） | 宏平均 F1 ≥ 85% |
| G3 | 按被试独立划分验证 | 训练集与测试集被试无重叠 |
| G4 | 代价敏感评估 | 跌倒漏检代价惩罚 ≥ 100 |

## 三、类别集定义

| 编号 | 类别名 | 说明 |
|------|--------|------|
| ω₁ | 跌倒 | 从站立到倒地的完整过程 |
| ω₂ | 走路 | 正常速度平地行走 |
| ω₃ | 跑步 | 中速跑步 |
| ω₄ | 静坐 | 静止坐在椅子上 |

## 四、风险/代价矩阵

跌倒漏检代价最高（C(跌倒→其他)=100），其他错误代价较低：

| 真实\预测 | 跌倒 | 走路 | 跑步 | 静坐 |
|-----------|------|------|------|------|
| 跌倒 | 0 | 100 | 100 | 100 |
| 走路 | 5 | 0 | 1 | 2 |
| 跑步 | 5 | 1 | 0 | 2 |
| 静坐 | 10 | 2 | 2 | 0 |

## 八、验收标准

1. **代码可复现**：`python main.py` 一键运行，输出结果与报告一致
2. **数据无泄漏**：训练/测试按被试划分，验证脚本通过
3. **代价敏感**：跌倒漏检代价 ≥ 100，结果中展示总代价
4. **文档完整**：README + 采集协议 + 实验记录
5. **可演示**：5分钟内展示从数据到结果的完整流程

## 九、风险预判

| 风险 | 可能性 | 应对措施 |
|------|--------|----------|
| 跌倒采集困难（安全风险） | 中 | 使用软垫、模拟跌倒动作 |
| 传感器数据漂移 | 低 | 定期校准、去零漂处理 |
| 被试数量不足 | 中 | 提前招募、组内互测 |
| 跌倒与跑步混淆 | 高 | 增加时域峰值特征、提高采样率 |
